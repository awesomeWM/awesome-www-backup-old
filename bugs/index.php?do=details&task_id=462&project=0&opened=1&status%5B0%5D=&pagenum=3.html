Details

I'm unable to generate any characters that are in the "second group" (i.e generated by holding Mode_Switch/ISO_Level3_Shift [Mod5] and the key) in awful.prompt.

For example pressing Mod5 + 's' produces an 's' although it should yield an 'ß' with my config.

xmodmap: up to 4 keys per modifier, (keycodes in parentheses):

shift Shift_L (0x32), Shift_R (0x3e)
lock
control Control_L (0x42), Control_R (0x6d)
mod1 Alt_L (0x40), Meta_L (0x9c)
mod2 Num_Lock (0x4d)
mod3 less (0x5e), ISO_Level3_Shift (0x71)
mod4 Super_L (0x25), Hyper_L (0x80), Super_L (0x73), Super_L (0x7f)
mod5 Mode_switch (0x5d), ISO_Level3_Shift (0x71), ISO_Level3_Shift (0x7c)

xorg.conf:
Section "InputDevice"
Identifier "Generic Keyboard"
Driver "keyboard"
Option "CoreKeyboard"
Option "AutoRepeat" "250 25"
Option "XkbRules" "xfree86"
Option "XkbModel" "thinkpad"
Option "XkbLayout" "us,de,de"
Option "XkbVariant" "altgr-intl,deadgraveacute,neo"
Option "XkbOptions" "ctrl:swapcaps,grp:sclk_toggle,grp_led:scroll,lv3:ralt_switch_multikey"
EndSection
