Details

I'm sorry if this is a duplicate with some other issue.

I have two monitors and a rule as follows:

{ rule = { class = "Pidgin" },
properties = { tag = tags[1][4], focus = false } },

This doesn't always work, i.e. the windows sometimes show on the second screen on whichever workspace is shown there instead of on the first screen, fourth workspace. To circumvent this I use:

{ rule = { class = "Pidgin" },
properties = { focus = false },
callback = function(c)
awful.client.movetotag(tags[1][4], c)
end },

This works but is ugly and chat windows show for a millisecond on my current active workspace which is annoying.

Am I doing something wrong or is this a bug?

Comment by Uli Schlachter (psychon) - Monday, 07 October 2013, 08:13 GMT
Could you try if this issue also appears without remborders?

The idea is: Pidgin starts without borders. Your rule moves it to (0, 0) of the second screen. Next, remborders adds borders to everything including pidgin. Now pidgin is placed at (-border_width, -border_width) which is outside of the screen. If this point happens to be on the other screen, well, pidgin is on that other screen. (Or if this isn't on any screen, the code will default to screen 1)

I am not really sure if this theory is sane. I am not even sure how setting the border_width changes the position and size of a client.

If this isn't due to remborders, I guess I'll have to make your config work locally and try this out myself. (If this is due to remborders: Uhm... At least we know how it happens, next we just need a fix)
