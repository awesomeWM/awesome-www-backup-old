Comment by Uli Schlachter (psychon) - Saturday, 21 May 2011, 13:45 GMT
I took a look at xvnc4viewer's source code. Fullscreen is implemented by making the window override-redirect and resizing it accordingly. Un-fullscreening is implemented by unsetting the override-redirect flag again.
awesome ignores both ConfigureNotify events which are generated by this. Meh.

In theory, after you pressed the fullscreen button (and nothing happened, still fullscreen), awesome's key bindings should work again. But I guess something weird which I didn't find yet makes this actually not work...

(I took a look at metacity's and dwm's source code. Metacity is complex and DWM doesn't handle ConfigureNotify events either)
