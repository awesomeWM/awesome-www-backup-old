Details

If you have focus on, say, screen 1, and on screen 2 (on active tag) client has been closed, focus forcefully switches to screen 2.

It is quite common if one runs big apps like virtualbox: it closes itself several seconds after instructed, in the meantime user could switch to other screen and get frustrated of being switched back. Another example is watching video with Mplayer while working on other screen: after movie has ended, Mplayer quits and drags focus.

Easily reproducible with opening two windows (1st and 2nd) on one screen, and killing 1st from terminal on other screen: 2nd window gets focus, while terminal should be.

I suspect bug lurks in awful.autofocus, in `check_focus` function: http://git.naquadah.org/?p=awesome.git;a=blob;f=lib/awful/autofocus.lua.in;h=bc00adda5b1cafc0f7a249e64c8a459b4d7995b3;hb=3.4#l18
Will try to investigate myself too.

Don't focus a different screen on unmanage ( FS#850 )

When some client on the left monitor was closed while client.focus is on the
right monitor (e.g. 'sleep 5 ; exit' in a terminal), awful.autofocus would shift
the input focus to whatever client happened to be next in the focus history on
the left monitor.

Fix this by only ever moving the input focus between screens when a tag is
selected, not when some client does its magic.

awful.autofocus: improve client focus with multiple screens

Changing screen focus to an empty tag leaves focus on a client on the
previous screen, if a tag with clients is focused next the focus
remains on the previous screen. Now awful.autofocus compares the tag
screen and clients screen to learn if the focus should be given to a
client on the focused screen. Client that gets the focus is selected
from history, as usual.
