Details

Since 3.5 yakuake doesn't work properly with awesome anymore. Yakuake is a quake-like drop-down terminal. In awesome <= 3.4 there already always was a flicker, when activating the yakuake window, while another window was present on the current tag, but in the end it all looked right (Window maximized below yakuake, yakuake in the foreground). Now it doesn't work anymore, there is a flicker, than yakuake AND the window both share half of the screen (as if yakuake wasn't floated), but the content of the both windows is not adapted (i.e. both "think" they are the only one).

My interpretation is that the window is not actually floated before it is drawn, but after, and that some goes wrong then.

This screencast should illustrate the issue:
http://soulrebel.in-berlin.de/awesome_yakuake.ogv

(it also illustrates a nother, unrelated issue, which I will open an extra bug report for)

I would be grateful for any help, patches or work-arounds, because it severely influences my work flow.

Thank you!
