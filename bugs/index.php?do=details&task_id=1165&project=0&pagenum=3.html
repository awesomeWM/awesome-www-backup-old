Comment by Mike Charlton (mikekchar) - Sunday, 01 September 2013, 09:14 GMT
I am not the original poster of the bug but I can verify that this bug exists in all of the above versions and also the github development version (a0e45e878e450d890a0ab541d48d8dd6c1ede943).

Reproduction steps:

1. Set up Awesome with Gnome on Ubuntu as per: http://awesome.naquadah.org/wiki/Quickly_Setting_up_Awesome_with_Gnome
2. Log in to Awesome-Gnome session
3. Open gnome-terminal
4. Press Mod4 key, hold it down and click with Button1 on the gnome-terminal with the mouse
5. Move the mouse with the above buttons held

Result:

Window resizes

Desired Result:

Window moves

Note: This happens the first time you try to move a gnome-terminal window and occasionally thereafter. Apart from the first time, the window usually moves as you would expect. Sometimes it resizes again, but I haven't been able to discover a pattern for where it happens.
