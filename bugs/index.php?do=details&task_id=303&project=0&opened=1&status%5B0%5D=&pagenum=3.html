Details

The prompt (modkey+F1) seems to use the US keyboard layout (I think) rather than the layout specified in xorg.conf:

Option "XkbLayout" "gb"

Steps to reproduce:
1. Start awesome (rc6) with the above keyboard layout.
2. Bring up the prompt (modkey+F1).
3. Try to press the '~' key and the '|' character appears in the prompt.

also, pressing '@' brings up '"' and vice versa, leading me to believe that the US layout is being used.

All other applications I have tested use the correct layout and I've never had a problem with things using the wrong layout in the past.

Comment by Konstantin Stepanov (kstep) - Sunday, 07 September 2008, 13:46 GMT
I guess the error is in encoding tables in keygrabber.c.

According to xev, keysym for small russian letter "a" is 0x06c1 (Cyrillic_a),
unicode for this symbol is 0xd0b0.

But according to this table:

0x0408, 0x0409, 0x040a, 0x040b, 0x040c, 0x0490, 0x040e, 0x040f, /* 0x06b8-0x06bf */
0x044e, 0x0430, 0x0431, 0x0446, 0x0434, 0x0435, 0x0444, 0x0433, /* 0x06c0-0x06c7 */
0x0445, 0x0438, 0x0439, 0x043a, 0x043b, 0x043c, 0x043d, 0x043e, /* 0x06c8-0x06cf */
0x043f, 0x044f, 0x0440, 0x0441, 0x0442, 0x0443, 0x0436, 0x0432, /* 0x06d0-0x06d7 */
0x044c, 0x044b, 0x0437, 0x0448, 0x044d, 0x0449, 0x0447, 0x044a, /* 0x06d8-0x06df */

keysym 0x06c1 is encoded as unicode 0x0430.

Looks like these tables were generated by perl with 'use utf8; use encoding "utf-8"',
which never returned correct unicode codes for me.
