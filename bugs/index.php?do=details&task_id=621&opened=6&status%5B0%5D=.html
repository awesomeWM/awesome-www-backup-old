Hi!

While it's good to have a sorted tab completion now (for programs, when using the "Run:" prompt), it seems to not be optimal.
It's slow to cycle between the available options to complete the entered chars.
For example, try "a" on the run prompt and then keep pressing TAB.
Awesome here takes 10% of the CPU (it sorts the available options every time a TAB is pressed?) and it's possible to see that there is a delay to switch to the next command.
Sometimes it seems that it's necessary to press TAB twice to pass to the next one.

Thank you!

Best regards,
Nelson
