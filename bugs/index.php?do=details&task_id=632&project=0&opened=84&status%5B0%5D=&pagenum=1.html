OK, technically this is not a bug. The flash fullscreen video has class Firefox-bin and instance firefox-bin. Going around the issue from the bug report is a matter of specifying both class and instance (or only instance if you choose so) for the Firefox window if using the tag property (for example: class = "Firefox", instance = "Navigator").

Going around the issue from the previous comment requires to create a rule for instance firefox-bin and set the floating property to true.
