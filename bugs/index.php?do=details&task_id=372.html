Details

I have a laptop with a 1024x768 internal LCD and I connect my 1440x900 external LCD.
I set them in a dual-screen configuration (xrandr --output VGA --above LVDS --mode 1440x900)

Now.. this is a simple PyQt4 application that tells me the available geometry on each of the screens. First this is what I get in single-screen scenario:

from PyQt4 import QtGui
app = QtGui.QApplication([])
app.desktop().availableGeometry(0)
## PyQt4.QtCore.QRect(0, 19, 1024, 749)

ie. the screen height - the height of the awesome statusbar (wibox)

This is what I get in the dual-screen scenario:
app.desktop().availableGeometry(0)
## I get PyQt4.QtCore.QRect(0, 0, 1440, 900)
app.desktop().availableGeometry(1)
## I get PyQt4.QtCore.QRect(0, 900, 1024, 768)

As you can see, it returns the whole height of the screen and doesn't substract the height of the statusbar (also, the top coordinate should be 19 and 919 respectively)

This seems to be not a bug, in fact.

Reading the source:

288 if ((d->screenCount == 1 || !isVirtualDesktop())
289 && X11->isSupportedByWM(ATOM(_NET_WORKAREA))) {
290 Atom ret;
[...]
295 e = XGetWindowProperty(X11->display,
296 QX11Info::appRootWindow(screen),
297 ATOM(_NET_WORKAREA), 0, 4, False, XA_CARDINAL,
298 &ret, &format, &nitems, &after, &data);
299
[...]
310 } else {
311 d->workareas[screen] = screenGeometry(screen);
312 }

The thing is that in your case, it take the else branch and ignore _NET_WORKAREA which is correctly set (run xprop and click on root window).

I mean: your screenCount is 2 so it does not pass and isVirtualDesktop() returns true if you use Xinerama, which you are right now, so this is not satsfied and the statement is false, so it will return you the screenGeometry.

The thing is that this Qt implementation is totally correct. In fact, no window manager which implements EWMH standard are supposed to handle Xinerama screen in a independant way like awesome does. So you're in fact reaching some sort of limitation on the standard implementation, because the _NET_WORKAREA is set for the workspaces belonging to a root windows. But in Xinerama you only have 1 root window, so you can only have one set of workspaces.

In this case, and due to this limitation 1) awesome only set _NET_WORKAREA for the first screen 2) Qt only knows about the workarea available for first screen in awesome, and for the whole display in any other wm.
