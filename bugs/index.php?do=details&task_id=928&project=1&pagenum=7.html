Details

I have a couple of applications which should have icons in the systray, including mail-notification (http://www.nongnu.org/mailnotify) and Spotify.
Spotify is started by my .xsessionrc file, while I started mail-notification manually from a terminal.

None of these applications icons, nor for other like the Debian update-notifier program, are shown in the systray. The icon for mail-notification was shown a little while, and then disappeared. Even if I restart the programs, if the icons are gone they will not show up again.

Comment by Uli Schlachter (psychon) - Wednesday, 28 September 2011, 15:40 GMT
Looks like this is a "meh, too lazy to implement the full spec"-problem. mail-notification hides/shows its tray icon via the following, which awesome just ignores:
15.654 000:<:05c6: 32: Request(18): ChangeProperty mode=Replace(0x00) window=0x0240000b property=0x17d("_XEMBED_INFO") type=0x17d("_XEMBED_INFO") data=0x00000001,0x00000001;

(Well, mail-notification also uses XUnmapWindow() / XWithdrawWindow() the first time that it wants to hide its tray icon. This is the only reason why hiding it works at all and it looks like the XEMBED specification doesnt mention this)
