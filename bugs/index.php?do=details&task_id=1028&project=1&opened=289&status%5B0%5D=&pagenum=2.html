Details

As seen by  FS#1019 , it would be nice if awesome were able to load SVGs.

Imlib2 (which we currently use for loading images) doesn't support SVGs. gdk-pixbuf does. Looks like all the other (important) image formats that Imlib2 supports can also be read by gdk-pixbuf. Also, gdk-pixbuf is smaller (measured by file size) since it does little more than just loading images while Imlib2 also provides quite some drawing API. Finally, there are gobject-introspection bindings for gdk-pixbuf, so lgi auto-magically provides bindings for it.

TL;DR: We should use gdk-pixbuf instead of imlib2.

(And annoy everyone by messing with our dependencies again ;-) )
