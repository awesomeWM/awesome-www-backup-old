Details

If you press Mod4+Enter to open a new terminal (urxvt in my case) in a floating layout, it opens a couple of pixels off the left edge of the screen.

This is particularly annoying when opening a terminal on the right-hand screen of a dual monitor set up, because this offset means the window's origin is located in the last few pixels of the left screen - so the terminal is on the right monitor but if you Mod4+F to make it full screen it goes full screen on the *left* monitor instead of the right one!

It would also be nice if it defaulted to appearing just below the wibox instead of over the top of it.

Comment by Adam Nielsen (Malvineous) - Friday, 12 February 2010, 09:25 GMT
Okay, now I feel stupid. Upon further investigation this was a leftover configuration issue from Compiz, which leaves the window border visible on maximised windows. Some years ago now I had set the offset to -2,0 for urxvt windows in ~/.Xresources to avoid this.

My apologies. At least it's nice to know that Awesome will let you shoot yourself in the foot if you want it to :-)
