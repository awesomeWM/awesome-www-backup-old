Details

Awesome handles _NET_CURRENT_DESKTOP via tag_view_only_byindex in the C code, but does not emit the "tag::history::update" signal, which is used by awful.tag to update/maintain its history.

I think the C code should emit this signal, to maintain the tag history properly.

My use case is using "rofi", which displays all apps from all tags, and goes to the one you select.

Here is rofi's code to switch to the target desktop:
https://github.com/DaveDavenport/rofi/commit/1834e25ef345fbe8fe26921c31b4c9f4a36d9723

Fix handling of _NET_CURRENT_DESKTOP messages ( FS#1219  , FS#1217  )

Instead of having the C code mess with which tags are selected, we now emit a
signal on the tag that says that something requested it to be selected. Lua can
then handle this by only switching tags on the correct monitor and by updating
the focus history correctly.
