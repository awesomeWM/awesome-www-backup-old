Comment by Julien Danjou (jd) - Wednesday, 27 May 2009, 15:03 GMT
Ok. I think that the bug is that when you set the focus to the client it is unbanned.

So, what may happens is:
- Set focus to the new client
- Move the client elsewhere
- Since the client has been moved, we receive a "mouse leave the newly created and moved window" event
- There is a window under us, we receive a "mouse enter the top left window" event
- Sloppy focus, so we give focus back to the top left window

So my patch is good, but not in the right place. I'll try to come with a more general fix.
