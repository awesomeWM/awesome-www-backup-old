Hi,

You probably seen the bug with Xephyr where some pixmap are drawn only on mouse over. Well, I found today that it is not a kdrive/Xephyr bug, but an awesome one. I switched to the native framebuffer driver on my mac because of some regression in the intel driver and now I got the same artifacts. Awesome 3.4 seem to work fine so do KDE and fluxbox, so it is really awesome not registering damaged areas in a fbdev compatible way. You might say "nobody use fbdev, you morron" (well, Linus Torvald would say that), but with EFI accelerated framebuffer, performance and power usage it actually rather nice, I think I will stick to it. The default config is not really usable because of this issue as the tags are never highlighted properly. So, if someone have an idea why it happen, it might be worth fixing after all.
