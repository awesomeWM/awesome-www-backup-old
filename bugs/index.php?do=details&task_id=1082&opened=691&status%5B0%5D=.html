Details

This has been mentioned in #awesome a few times, but there doesn't seem to be a report yet.

When resizing windows in floating layout (mod+rightclick), the resizing handle does not go away when I unclick. The window continues to be in resizing mode until I leftclick:

1) mod+rightclick
2) move the mouse - window size changes as expected
3) let go of mod and rightclick**
4) (BUG) resizing handle doesn't go away - window still gets resized
5) leftclick to finally leave resizing mode

** Not sure if the order matters here. Unforunately this happens intermittently so it's hard to test. I can confirm that this happened when I let go of mod before unclicking, but I can't confirm if this can also happen if I unclick before unmodding. Most of the time when it happens, I've let go of both of them too close together to tell which was first.

Seeing this behavior in 3.5 and git.

Revert "Support more than 5 mouse buttons" ( FS#1082  )

This reverts commit bd8158495e059baa0e8ba525b813254451447f4a .

The idea was to track the current list of pressed and depressed mouse buttons,
because we get button events for more than 5 buttons, but can only query the
state of the first 5 buttons.

However, there are cases where we see button presses, but won't see the
corresponding release event. This is quite bad.
