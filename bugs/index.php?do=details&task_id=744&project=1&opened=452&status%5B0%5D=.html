FS#744 - Don't strip off the RPATH

Details

Cmake strips off the RPATH of the binary by default. On Solaris, this is unacceptable, because the binary must be able to find libraries from /opt/csw/lib, and there is no /etc/ld.so.conf on Solaris, so RPATH is the only option.

The following patch allows the binary to work:

https://gar.svn.sourceforge.net/svnroot/gar/csw/mgar/pkg/awesome/trunk/files/turn-off-rpath-breakage.patch

See also:
http://ptribble.blogspot.com/2010/01/stupid-build-systems.html
