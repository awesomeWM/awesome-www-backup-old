Comment by Gregor Best (farhaven) - Tuesday, 27 April 2010, 21:26 GMT
The tasklist uses the flex layout by default (just as the old one pre-widget layouts did). The flex layout uses all space not occupied by other layouts, thus it makes most sense to have it as the last active layout. I'm not quite sure why you think the default rc is 'bad' or 'poor' as documentation, because it shows how a sample widget box is done in the wild, and it is always up-to-date.
