Details

When using nvidia ViewPort settings, awesome uses the whole xrandr "output" resolution instead of xrandr "screen" (ViewPortIn) resolution. So the actual desktop size is higher than monitor resolution.

Example:
ViewPort metamode:
Option "metamodes" "TV: 720x576 { ViewPortOut=640x519+44+26, ViewPortIn=600x450 }"
xrandr:
Screen 0: minimum 8 x 8, current 600 x 450, maximum 8192 x 8192
...
TV-0 connected primary 720x576+0+0 (normal left inverted right x axis y axis) 0mm x 0mm

Here, awesome desktop resolution is 720x576 instead of 600x450.

However e16 window manager sets desktop resolution correctly to 600x450.

tested with:
awesome 3.5
nvidia driver 313.18
xorg server 1.13.2
