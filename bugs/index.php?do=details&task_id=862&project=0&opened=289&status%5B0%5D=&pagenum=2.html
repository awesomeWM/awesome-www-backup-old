Details

urxvt's true trasnparency support (compositing, not pseudo-transparency) is broken in current versions of awesome-git.
Forced transparency (c.opacity = 0.7) works, but urxvt's internal background-only transparency does not.

Steps to test:

Have urxvt configured with these two lines in .Xresources/.Xdefaults:
URxvt.depth: 32
URxvt*background: rgba:0000/0000/0000/cccc

URxvt.transparent must NOT be set, or must be set to false.

Start awesome
start a compositing manager. Tested with xcompmgr and dcompmgr
start urxvt
Check if transparency is functioning (is the wallpaper visible through the window? Are other windows visible below when floating?)

The exact commit that broke it is:

102063dbbdfb0bc9f43268d98f7dcb5269547395 is the first bad commit
commit 102063dbbdfb0bc9f43268d98f7dcb5269547395
Author: Uli Schlachter <psychon@znc.in>
Date: Fri Jul 30 20:54:20 2010 +0200

Turn this into a reparenting WM

Each window is now reparented under a window created by awesome. This window is
NOT visible, this commit should not cause any visible differences. Instead, this
is done to work around problems with X11.

The only alternative would be to ban windows by moving them offscreen which is
ugly in itself and breaks pagers. Doing it like this seems to be the better
solution.

Signed-off-by: Uli Schlachter <psychon@znc.in>
