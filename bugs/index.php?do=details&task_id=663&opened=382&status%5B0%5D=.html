Details

After upgrade from 3.3.x to 3.4 I noted a regression. I'll try to describe all steps to reproduce this:

1. Configure tag 3 to use floating layout

awful.layout.set(layouts[12], tags[s][3])

2. Configure firefox (or other application) to open on tag 3

{ rule = { class = "Firefox" },
properties = { tag = tags[1][3] } },

3. Go to tag 3
4. Open firefox
5. Resize it (with mouse) to use all available space, don't maximize it, the window could not have floating property, just the tag has it.
6. Close Firefox
7. Reopen it and note the window will use the last size you set, using all available space on the tag
8. Close Firefox one more time
9. Move to another tag using tile layout
10. Open a xterm
11. Open firefox

You will note that firefox window will start on current tag, resizing xterm for a second and after it's moved to tag 3

Now if you go to tag 3, firefox' size is using half of screen, the same if you open it on tiled tag and have a xterm already opened.

Please let me know if you need more data to check this problem, i'm attaching my rc.lua too, maybe it can help to reproduce.

I don't know if it's related, but another small thing that i noted is when I have firefox opened on my tag 3 (floating layout), if firefox need to open a small window (save as, confirmation, download), or a new browser window (ctrl+N), the new firefox window doesn't get the focus. All other new windows recieve focus when is opened, just firefox ones had these problems.

For the records, i'm using firefox 3.0.x on FreeBSD 9.0-CURRENT
