Hi,
I'm using awesome 2.3 (this also happened on earlier versions/snapshots) and I noticed a specific bug with gmplayer. The problem is: while viewing video files if I attempt to make the video window full screen gmplayer freezes completly. The player doesn't freeze in other circumstances (non-fullscreen), or if I stretch the window to a full screen size.

Important things to note are:
- there is no freeze if I start mplayer and then go full screen, bug is present only with the gui
- this doesn't happen under other WM's I have on my system (fvwm, kwin)

Arch GNU/Linux, xorg-server 1.4.0.90, graphics from intel with xf86-video-intel 2.2.1,
mplayer 1.0rc2 with xv/X11 video output (double buffering, direct rendering... makes no difference)
