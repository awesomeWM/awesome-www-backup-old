client.content: Use correct client size ( FS#1150  )

The size saved in c->geometry also includes the titlebars. Thus, for getting the
window content, we have to subtract this from the size.

Before this, the call to xcb_image_get() was failing with a BadMatch error,
because we were asking for an area that is outside of the actual client's
window's geometry.

Comment by Uli Schlachter (psychon) - Monday, 15 July 2013, 21:23 GMT
Idea:
The code currently looks like this:

xcb_image_t *ximage = xcb_image_get(globalconf.connection, c->window, 0, 0, c->geometry.width, c->geometry.height, ...);

However, c->geometry describes the geometry including the titlebars and thus is not the "real" size of the client window. According to http://tronche.com/gui/x/xlib/graphics/XGetImage.html:

"the specified rectangle of the window would be fully visible on the screen and wholly contained within the outside edges of the window, or a BadMatch error results."

Thus, it seems to me that we are asking for a too large area and thus get back a BadMatch error.

So far this is just a theory which would need to be tested. Ugly hack for testing would be to change c->geometry.height into c->geometry.height - c->titlebar[CLIENT_TITLEBAR_TOP].size (which should work assuming that you only have a titlebar at the top and no other titlebars).
Alternative, someone could this this with titlebars disabled.
