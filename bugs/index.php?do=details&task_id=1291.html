Details

Using awesome 3.5.5, in Arch Linux, in VirtualBox

Steps:
1. Start awesome, using default rc.lua from /etc/xdg/awesome/rc.lua
2. In tag 1, open a terminal
3. Switch to tag 2
4. Switch back to tag 1

Problem:
The display switches back to Tag 1's contents, but Tag 2 remains highlighted. There's a little ghost of a highlight under tag 1. The taskbar also continues to show tag 2's contents (blank, in the case of the above example).
The tag and taskbar display can be temporarily corrected by opening another window or changing the layout.
Problem does not happen when switching between tags that have no open windows.

I first noticed this after updating to xorg-server 1.16, which I'm guessing has something to do with it. When I updated my system that day, the following updates also happened:
cryptsetup (1.6.5-1 -> 1.6.5-2)
libdrm (2.4.54-1 -> 2.4.55-1)
libxfont (1.4.7-3 -> 1.5.0-1)
linux (3.15.5-2 -> 3.15.7-1)
virtualbox-guest-modules (4.3.14-1 -> 4.3.14-3)
libepoxy (1.2-2)
xorg-server-common (1.15.2-1 -> 1.16.0-5)
xf86-input-evdev (2.9.0-1 -> 2.9.0-2)
xorg-server (1.15.2-1 -> 1.16.0-5)
virtualbox-guest-utils (4.3.14-1 -> 4.3.14-2)

I also posted this in the Arch forums, one other person confirmed it.
https://bbs.archlinux.org/viewtopic.php?id=185020
