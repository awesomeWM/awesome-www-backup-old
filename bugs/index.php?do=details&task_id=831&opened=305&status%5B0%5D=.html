Details

All our visible windows now use an ARGB visual. The only exception to this is the systray window which can't do this yet because we have to reparent other windows to it.

However, the system tray specs allow ARGB systrays:

"If the _NET_SYSTEM_TRAY_VISUAL property (see below) is present, tray icon windows should be created using that visual. If the property is not present, then tray icon windows should be created using the default visual of the screen."

The problem with this is:

"If this property is set to a visual with an alpha channel, the tray manager must use the Composite extension to composite the icon against the background using PictOpOver."

So I guess we should check for the COMPOSITE and RENDER extensions and only if both are available shall we use a 32bit visual for the systray. Meh. :(
