It could be me doing something wrong, but I don't think the progressbar should behave in this manner. By default the progressbar's height gets set to the height of the wibox, so I tried the following:

cpuwidget = awful.widget.progressbar()
cpuwidget:set_height(4)
...
right_layout:add(cpuwidget)

The progressbar's height got set to the height of "mywibox" regardless. Setting the height arg in the progressbar constructor doesn't work either.

Comment by Uli Schlachter (psychon) - Wednesday, 12 December 2012, 16:58 GMT
Sorry, but this is by design. The fixed.horizontal layout ignores the height that a widget wishes to have (and fixed.vertical ignores its width) and just forces it to full height/width, because otherwise text wouldn't be properly centered vertically. The text doesn't need all the height of the wibox and this caused it to always be aligned to the top.

The "proper" way would be to add some margin around the widget (wibox.layout.margin), I guess. Is that enough for you or do we need to come up with something new?
(Also, do you really want a 4px progressbar? In 3.4 that was just a progressbar with 4 possible states and that's quite small)
Comment by Björn Åström (bjoast) - Wednesday, 12 December 2012, 18:36 GMT
I thought I would have to use margins, and I guess it makes sense too considering the layout :-). And yes, I want a 4 pixel progressbar since I am using it in horizontal mode, so the number of states is limited by its width, which is 100 by default.

Using the following code I can modify the height of the progressbar by changing the margins:

cpuwidget = awful.widget.progressbar()
cpumargin = wibox.layout.margin(cpuwidget, 0, 0, 15, 15)
...
right_layout:add(cpumargin)

I tried using a fixed vertical layout and the height arg, while putting a textbox below the progressbar, and with some trickery I managed to center it and it looks fine in my 48 pixel top-wibox:

cpuwidget = awful.widget.progressbar({ height = 4 })
cputext = wibox.widget.textbox()
cputext:set_text("CPU")
cpulayout = wibox.layout.fixed.vertical()
cpulayout:add(cpuwidget)
cpulayout:add(cputext)
cpumargin = wibox.layout.margin(cpulayout, 0, 0, 18)
...
right_layout:add(cpumargin)

So this was not bug, it was me being stupid. You may close this task. :-)
