Hi,

for some reason, when I start "rdesktop" or "xfreerdp", the window itself appears on the second screen (which is what I want) but the task is placed in the tasklist of screen 1.
It doesn't matter whether I start it on screen 2 or specify a rule to start it on screen 2. The windows is there but the task is always placed onto screen 1. So when I switch between tags on screen 1 the application window disappears as well. What helps is modkey+o.
I can reproduce it with awesome 3.4.x and current 3.5.
With other programs, Terminals, Browser and so on it works just fine.

Just in case, I use Nvidia's TwinView with two screens.
