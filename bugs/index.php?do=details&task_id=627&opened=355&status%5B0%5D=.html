Details

Using a default install of 3.3.4 on a laptop with an azerty keyboard: all combo involving numeric keys are broken.
Namely:
Mod4 + 1-9 : Switch to tag 1-9.
Mod4 + Shift + 1-9 : Tag client with tag.
Mod4 + Control + 1-9 : Toggle tag view.
Mod4 + Shift + Control + 1-9 : Toggle tag on client.

It makes Awesome barely usable for azerty users: French and Belgium mainly.

There was a thread in the mail list: http://article.gmane.org/gmane.comp.window-managers.awesome/4914
"Jérôme M. Berger" proposed an easy solution: uses the concatenation "#1"..i-1 instead of just i. Since "#10" to "#19" are the physical key coordinates, it works on azerty and qwerty.

I don't provide a patch, but the fix should be easy, just replace i by "#1"..i-1 in awesomerc.lua between lines 241 and 261 around here:

http://git.naquadah.org/?p=awesome.git;a=blob;f=awesomerc.lua.in;h=7c0360be238aa21c38d24f605f62ef6334e7274b;hb=HEAD#l241

C'mon, I was kiding.

The main problem is that I've got the feeling that if we start to use keycode, we are losing the sense of keybindings. If we bind to #10 to #19 or whatever keycodes are for these physical days, we are bounding to physical keys and not to keysym anymore, which is a totally different concept.

There's also a change that i.e. Mod4+j does not work by default because j is not directly available on a layout someone is using. So I agree to be totally pragmatic and make things works, but the proposed solution is to bind to physical keys. If we go this way, let's go it totally, but it's gonna be messy.
