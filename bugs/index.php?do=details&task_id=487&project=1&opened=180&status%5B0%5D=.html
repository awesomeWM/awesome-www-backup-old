Details

The error is simple enough -- if you put something into fullscreen (or maximize) and it's supposed to not honour size hints, it does anyway.

Reproduce:
- Make clients disable size hints on start (in your hook)
- fullscreen a hinted client (like a terminal)
- see gaps around edges (lets hope your screen isn't perfectly the right size :)

This seems to be a bit more of a bug than I hoped -- examination of the code didn't reveal any problems to me. The sightly funny thing is that:
- client_setfullscreen seems to behave properly. It gets the correct geometry for the new window.
- client_setfullscreen also properly, statically disables size hints for client_resize: client_resize(c, geometry, false);
- client_resize seems to do "the right thing" as far as I can tell, but the problem may be there.

Anyway, thanks.
