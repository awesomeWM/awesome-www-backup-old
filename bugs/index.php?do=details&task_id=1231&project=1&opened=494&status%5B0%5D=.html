Comment by Daniel Hahler (blueyed) - Friday, 14 March 2014, 19:55 GMT
I have tracked it down to how I start `gnome-settings-daemon`.

When starting it from rc.lua, awesome does not pick up the changes done through it (Display setup from Unity/Ubuntu).

If I call it before `exec awesome` it requires a sleep statement (I have only tried 5 seconds), and when starting it via ~/.config/autostart as follows, it works (copied from /etc/xdg/autostart, with OnlyShowIn commented):

cat ~/.config/autostart/gnome-settings-daemon.desktop
[Desktop Entry]
Type=Application
Name=GNOME Settings Daemon
Exec=/usr/lib/gnome-settings-daemon/gnome-settings-daemon-localeexec
# OnlyShowIn=GNOME;Unity;
NoDisplay=false
X-GNOME-Autostart-Phase=Initialization
X-GNOME-Autostart-Notify=true
X-GNOME-AutoRestart=true
X-Ubuntu-Gettext-Domain=gnome-settings-daemon
X-GNOME-Autostart-enabled=false
