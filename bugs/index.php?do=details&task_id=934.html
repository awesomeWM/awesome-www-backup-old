Details

i found a little edge case with sticky windows.

when a window instantiate with a sticky flag and when i remove the flag then i'm landing on no tag, not focusing the window anymore and can't see it anymore. so it seems that the window is not attached to a tags and when it loses the sticky flag it can't be rendered because there is no tag that could toggle visibility (this is just my conclusion of watching the behavior).

currently i only know one window which starts with a sticky flag and it's the "Document Modified on Disk" notification of kate 3.6.5 (it's enabled by a flag under "General Options" called "Warn about files modified by foreign processes").

Tagging sticky clients shouldn't hurt anything, but it helps a lot if the client
later becomes unsticky. I'd have no tags in that case.
