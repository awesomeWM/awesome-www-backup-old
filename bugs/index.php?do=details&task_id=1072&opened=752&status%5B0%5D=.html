Details

I have the following problem in a standard install (standard rc.lua).

1) Start awesome
2) press ctrl-p to invoke menubar
3) type "firefox"
4) press return

then menubar disappears but firefox does not start (as should be intended).

This seems to be a strange bug because 1)-4) works as intended if I "restart awesome" and do everthing in the same way again.

I noticed that with other situations as well, that some functionality is only present after running rc.lua the second time.

Comment by Uli Schlachter (psychon) - Tuesday, 01 January 2013, 15:24 GMT
Ok, then this bug is easy to fix (thanks for the pointer):

awful.prompt handles Return and KP_Enter in the same way (and some other keys, e.g. ctrl-j and ctrl-k).

menubar is missing KP_Enter and should definitely treat it the same way as Return.
Comment by Bernhard Koenig (Antunes) - Tuesday, 01 January 2013, 16:07 GMT
Good idea. The same holds for the terminal shortcut "mod + Return", by the way. It does not work with "mod + KP_Enter".

This is usually not a problem, but some laptops do not have a return key (because normal return is mapped to KP_Enter).
