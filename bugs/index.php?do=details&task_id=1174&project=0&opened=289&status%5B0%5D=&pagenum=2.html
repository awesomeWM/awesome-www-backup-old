Comment by Uli Schlachter (psychon) - Friday, 04 October 2013, 16:55 GMT
Meh. The tooltip positioning only ever worked for the wibox and was always broken everywhere else. The code tries to place the tooltip directly (centered) below the mouse pointer and then expects awful.placement.no_offscreen to fix things up so that the tooltip isn't actually below the mouse pointer.

So either we need a more intelligent placement strategy or come up with some ideas so that tooltips aren't immediately hidden on mouse::enter.

Here is how gtk places tooltips: https://git.gnome.org/browse/gtk+/tree/gtk/gtktooltip.c?id=e61c1245096e46fda3385afa5c0dcd8e3574d51f#n1084
