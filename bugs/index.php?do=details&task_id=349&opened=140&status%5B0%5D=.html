In patch:

+ #ifndef __FreeBSD__
csfd = socket(AF_UNIX, SOCK_SEQPACKET, 0);
+ #else
+ csfd = socket(PF_UNIX, SOCK_STREAM, 0);

AF_UNIX => PF_UNIX (or even PF_LOCAL, which is better according "man socket", as PF_UNIX is deprecated),
SOCK_SEQPACKET => SOCK_STREAM.

And here:

+ #ifndef __FreeBSD__
if(send(csfd, msg, msg_len, MSG_NOSIGNAL | MSG_EOR) == -1)
+ #else
+ if(send(csfd, msg, msg_len, MSG_NOSIGNAL | MSG_EOF) == -1)
+ #endif

MSG_EOR => MSG_EOF,
there's "illegal parameters" error when send()'ing.

Sorry, I didn't answer later, I was very busy during this week.

I wrote a simple test program:

#include <stdio.h>
#include <sys/types.h>
#include <sys/socket.h>
#include <errno.h>

int main (int argc, char* argv[])
{
int sh,proto;
for (proto = 0; proto < 259; proto++) {
sh = socket(PF_LOCAL, SOCK_SEQPACKET, proto);
printf("proto = %d, socket() == %d, error = %s\n", proto, sh, strerror(errno));
close(sh);
}
}

All socket() calls with any of protocol number resulted in "Protocol not supported error.",
so I can conclude SOCK_SEQPACKET is *not* supported for PF_LOCAL protocol family:

proto = 0, socket() == -1, error = Protocol not supported
proto = 1, socket() == -1, error = Protocol not supported
proto = 2, socket() == -1, error = Protocol not supported
proto = 3, socket() == -1, error = Protocol not supported
...
proto = 257, socket() == -1, error = Protocol not supported
proto = 258, socket() == -1, error = Protocol not supported
