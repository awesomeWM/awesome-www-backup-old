Details

I'm running awesome 3.4.4 on gentoo.

The prompt shows inconsistent behavior with paths that need to be escaped:
While a normal path is completed properly:

vim ne->vim newfolder/

a filename containing spaces is expanded without trailing slash:

vim ne->vim new\ folder

a filename containing $ is not escaped and has no trailing slash:

vim ne->vim new$folder

and a filename containing @ is not escaped, but completed including slash:

vim ne->vim new@folder/
